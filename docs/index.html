glab CLI

Work seamlessly with GitLab from the command line. Manage merge requests, issues, pipelines, and more.

Get Started View on GitHub
brew tap PhilipKram/tap https://github.com/PhilipKram/Gitlab-CLI && brew install PhilipKram/tap/glab

Why glab?

Merge Requests

Create, review, approve, and merge MRs without leaving your terminal. Supports draft MRs, assignees, reviewers, and labels.

glab mr create --title "Feature" --draft

Issues

Full issue lifecycle management. Create, assign, label, comment, and close issues from the command line.

glab issue list --state opened

CI/CD Pipelines

Trigger pipelines, view job logs, retry failed builds, and monitor CI status right from your terminal.

glab pipeline run --ref main

Repositories

Clone, create, fork, and manage repositories. Full support for groups and namespaces.

glab repo clone owner/repo

Familiar Interface

Follows the same command patterns as GitHub CLI (gh). If you know gh, you already know glab.

glab <noun> <verb> [flags]

OAuth & Self-Hosted

Login interactively via browser OAuth or personal access tokens. Works with gitlab.com and any self-hosted instance.

glab auth login

Authentication

After installing, authenticate with your GitLab instance. OAuth is the default — just run glab auth login and it opens your browser:

# OAuth login (default) — opens your browser
glab auth login

# First run prompts for host, protocol, and OAuth app ID.
# Subsequent runs remember everything and go straight to the browser.

# Login to a specific host
glab auth login --hostname gitlab.example.com

# Login with a personal access token instead
glab auth login --token glpat-xxxxxxxxxxxxxxxxxxxx

# Or set an environment variable
export GITLAB_TOKEN="glpat-xxxxxxxxxxxxxxxxxxxx"

Required token scopes: api, read_user, write_repository

OAuth Setup

Before your first OAuth login, create an OAuth application in your GitLab instance under Settings > Applications:

  • Redirect URI: http://localhost:7171/auth/redirect
  • Scopes: api, read_user, write_repository, openid, profile

The CLI starts a local server on port 7171, opens your browser for authorization, and exchanges the code for a token using PKCE — no client secret needed. Your OAuth application ID is saved automatically so you only enter it once.

Per-Host Configuration

OAuth settings are saved automatically during login. You can also set them manually: 

# Store OAuth client ID for a self-hosted instance
glab config set client_id <app-id> --host gitlab.example.com

# Store custom redirect URI (default: http://localhost:7171/auth/redirect)
glab config set redirect_uri http://localhost:8080/callback --host gitlab.example.com

# Store custom OAuth scopes
glab config set oauth_scopes "api read_user write_repository" --host gitlab.example.com

Command Reference

Global Flags

--repo, -RSelect a GitLab repository using HOST/OWNER/REPO format (e.g., -R gitlab.example.com/group/project)

When no --repo is specified, glab resolves the host from the git remote. If the remote isn't a GitLab host, it falls back to the default host, then to the first authenticated host.

Examples

Complete MR Workflow

# Create a merge request from current branch
glab mr create --title "Add user authentication" \
  --description "Implements OAuth2 login flow" \
  --assignee @alice --reviewer @bob \
  --label feature,auth --draft

# List open MRs assigned to you
glab mr list --state opened

# Check out a teammate's MR for review
glab mr checkout 42

# Leave an inline comment on the diff
glab mr comment 42 --body "Consider refactoring this" \
  --file "src/auth.go" --line 57

# Approve and merge
glab mr approve 42
glab mr merge 42 --squash --remove-source-branch

CI/CD Pipeline Monitoring

# Trigger a pipeline on main
glab pipeline run --ref main --variables DEPLOY_ENV=staging

# Watch pipeline status
glab pipeline list --ref main --limit 5

# Debug a failed job
glab pipeline jobs 12345
glab pipeline job-log 67890

Issue Tracking

# Create a bug report
glab issue create --title "Login page broken on Safari" \
  --label bug,P1 --assignee @alice

# Find issues by label
glab issue list --label bug --state opened

# Close with a comment
glab issue comment 42 --body "Fixed in !99"
glab issue close 42

Working with Remote Projects

# Target a project on any GitLab instance (no git clone needed)
glab issue list -R gitlab.example.com/group/project
glab mr list --state opened -R gitlab.example.com/team/app

# Direct API access with specific host
glab api '/projects?membership=true' --hostname gitlab.example.com

Upgrading

# Check if a newer version is available
glab upgrade --check

# Upgrade to the latest version
glab upgrade

# Skip confirmation prompt
glab upgrade --yes

For binary installs, glab upgrade downloads the latest release with checksum verification and replaces the binary in-place. Homebrew and package manager installs are detected automatically and print the appropriate upgrade command instead. A startup banner also notifies you when a new version is available — the check runs in the background and never slows down your commands.

JSON Output & Scripting

# Get MR data as JSON
glab mr list --json | jq '.[].title'

# Direct API access for anything not covered
glab api projects --method GET
glab api graphql --method POST \
  --body '{"query":"{ currentUser { name } }"}'

Configuration

Configuration is stored in ~/.config/glab/. Override with the GLAB_CONFIG_DIR environment variable.

Global Keys

editorPreferred text editor
pagerPreferred pager program
browserPreferred web browser
protocolGit protocol: https (default) or ssh
git_remoteDefault git remote name (default: origin)

Per-Host Keys (use with --host)

client_idOAuth application ID
redirect_uriOAuth redirect URI (default: http://localhost:7171/auth/redirect)
oauth_scopesOAuth scopes (default: openid profile api read_user write_repository)
protocolGit protocol for this host
api_hostAPI hostname override

Environment Variables

GITLAB_TOKENAuthentication token (overrides stored config)
GLAB_TOKENAuthentication token (alternative)
GITLAB_HOSTDefault GitLab hostname (default: gitlab.com)
GLAB_CONFIG_DIRConfiguration directory path
